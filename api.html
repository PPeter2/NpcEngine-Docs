API Reference

Complete reference for every public method in NPCEngine. All methods are available on the object returned by Engine.new() unless noted otherwise.

Engine.new()
Constructor

Creates and returns a new NPC engine instance. Wires together all sub-systems: FSM, Pathfinder, LODBrain, Blackboard, Perception, AggroTable, ContextSteering, Network, AnimationController, DebugOverlay, Dialogue, BehaviourTree, and UtilityAI.

NPCEngine.new(config: NPCConfig) → NPCEngineInstance
FieldTypeRequiredDescription
ModelModel✓The NPC model. Must contain a Humanoid and HumanoidRootPart.
MoveSpeednumber?—Walk speed. Default 16.
SleepRangenumber?—Distance beyond which the NPC fully sleeps. Default 100. Set to 9999 to never sleep.
NetworkIdstring?—Unique identifier for this NPC's network channel. Auto-generated if omitted.
 
local npc = NPCEngine.new({
    Model = workspace.MyNPC,
    MoveSpeed = 16,
    SleepRange = 120,
    NetworkId = "Guard_01",
})
State Machine
RegisterState()
FSM

Declares a named state with lifecycle callbacks. Must be called before ChangeState can transition to it.

npc:RegisterState(stateName: string, callbacks: StateCallbacks)
CallbackWhen it fires
OnEnterImmediately when the state becomes active.
OnUpdateEvery LOD brain tick while the state is active.
OnExitBefore the state is replaced by a new one.
 
npc:RegisterState("Chase", {
    OnEnter = function()
        print("Chasing!")
    end,
    OnUpdate = function()
        local target = npc:GetAggroTarget()
        if not target then npc:ChangeState("Idle") end
    end,
    OnExit = function()
        npc:StopMovement()
    end,
})
ChangeState()
FSM

Transitions to a registered state. Fires the current state's OnExit, atomically cleans up the state's sub-Janitor, then fires the new state's OnEnter. Blocked during active crowd control.

npc:ChangeState(stateName: string)
Throws if stateName has not been registered with RegisterState.
npc:ChangeState("Idle")
BindBehavior()
FSM

Automatically triggers a state transition when a signal fires. The connection is tracked by the master Janitor and cleaned up on Destroy().

npc:BindBehavior(triggerSignal: Signal, targetState: string)
-- Transition to Alert whenever the alarm fires
npc:BindBehavior(AlarmSystem.OnAlarm, "Alert")
Blackboard
BlackboardSet()
Blackboard

Stores a value in the NPC's shared context store. Readable by any state, behaviour tree node, or utility AI action on this NPC instance.

npc:BlackboardSet(key: string, value: any)
npc:BlackboardSet("target", somePlayer)
npc:BlackboardSet("isAggressive", true)
npc:BlackboardSet("lastDamageTime", os.clock())
BlackboardGet()
Blackboard

Reads a value from the blackboard. Returns nil if the key has not been set.

npc:BlackboardGet(key: string) → any
local target = npc:BlackboardGet("target") :: Player?
if target then
    print("Current target:", target.Name)
end
Perception
ConfigurePerception()
Perception

Sets ranges and parameters for all three senses. Can be called at any time including after construction to update live values.

npc:ConfigurePerception(config: PerceptionConfig)
FieldTypeDefaultDescription
sightRangenumber?60Maximum sight distance in studs.
sightFOVnumber?90Field of view angle in degrees. Use 360 for full awareness.
hearingRangenumber?30Hearing radius. Not blocked by walls unless hearingWalled is true.
smellRangenumber?10Smell radius. Never blocked by walls.
memoryTimenumber?8Seconds before a detected target is forgotten after leaving range.
 
npc:ConfigurePerception({
    sightRange = 80,
    sightFOV = 110,
    hearingRange = 40,
    smellRange = 8,
    memoryTime = 10,
})
IsAwareOf()
Perception

Returns true if a player is currently in the NPC's perception memory.

npc:IsAwareOf(player: Player) → boolean
if npc:IsAwareOf(somePlayer) then
    npc:AddThreat(somePlayer, 50)
end
GetLastKnownPosition()
Perception

Returns the last known world position of a player from perception memory. Returns nil if the player has never been detected or memory has expired.

npc:GetLastKnownPosition(player: Player) → Vector3?
local lkp = npc:GetLastKnownPosition(target)
if lkp then
    npc:PathToAsync(lkp):catch(function() end)
end
GetPerceivedTargets()
Perception

Returns an array of all players currently in perception memory.

npc:GetPerceivedTargets() → { Player }
Aggro Table
AddThreat()
Aggro

Adds threat to a player in the aggro table. If the player is not already in the table they are added. Threat decays over time when the player leaves aggro range.

npc:AddThreat(target: Player, amount: number)
-- Add 100 threat when player hits the NPC
npc:AddThreat(attackingPlayer, 100)
RemoveThreat()
Aggro

Removes a player from the aggro table entirely.

npc:RemoveThreat(target: Player)
Taunt()
Aggro

Forces the NPC to target a specific player for a duration, regardless of threat levels. The taunted player becomes the priority target and threat decay is blocked during the taunt window.

npc:Taunt(target: Player, duration: number)
-- Tank ability: force boss to target me for 5 seconds
npc:Taunt(tankPlayer, 5)
GetAggroTarget()
Aggro

Returns the current highest-threat player, or nil if the aggro table is empty.

npc:GetAggroTarget() → Player?
Context Steering
SteerToward()
Steering

Activates context steering toward a target position. Uses 16-slot interest and danger maps. Raycasts are halved per tick via alternating slot cache for performance. Use for close-range movement where smooth avoidance matters more than path accuracy.

npc:SteerToward(targetPosition: Vector3)
-- Use steering when close, pathfinding when far
local dist = (root.Position - myRoot.Position).Magnitude
if dist < 15 then
    npc:SteerToward(root.Position)
else
    npc:PathToAsync(root.Position):catch(function() end)
end
SteerStop()
Steering

Clears the steering interest target and stops steering movement.

npc:SteerStop()
AddSteeringDanger()
Steering

Registers a dynamic danger source that the steering system will avoid. Weight 0–1 controls how strongly the source repels movement.

npc:AddSteeringDanger(part: BasePart, weight: number)
-- Make NPCs avoid each other
for _, otherNPC in allNPCs do
    npc:AddSteeringDanger(otherNPC._rootPart, 0.8)
end
Navigation
PathToAsync()
Navigation

Computes and executes a path to a world position. Returns a cancellable Promise. Handles jump waypoints, stuck detection with automatic recompute, and StreamingEnabled safety.

npc:PathToAsync(targetPosition: Vector3) → Promise
If the target hasn't moved more than 4 studs from the last computed destination, the existing promise is returned instead of recomputing.
npc:PathToAsync(targetPosition)
    :andThen(function()
        print("Arrived!")
    end)
    :catch(function(err)
        warn("Path failed:", err)
    end)
StopMovement()
Navigation

Immediately cancels the active navigation promise and stops the humanoid. No physics stutter.

npc:StopMovement()
PredictTargetPosition()
Navigation

Returns where a moving BasePart will be in leadTime seconds based on its current linear velocity.

npc:PredictTargetPosition(targetPart: BasePart, leadTime: number) → Vector3
-- Path to where the player will be in 0.3 seconds
local predicted = npc:PredictTargetPosition(playerRoot, 0.3)
npc:PathToAsync(predicted):catch(function() end)
IsInLineOfSight()
Navigation

Returns true if a BasePart is within the NPC's FOV angle and has a clear raycast path.

npc:IsInLineOfSight(targetPart: BasePart, fovAngle: number, maxDistance: number) → boolean
ApplyCrowdControl()
CC

Forcefully interrupts AI execution and applies a physical effect for a duration. Restores the previous state when expired. All four CC types clean up via Janitor — zero residual effects.

npc:ApplyCrowdControl(ccType: CrowdControlType, duration: number, knockbackDir: Vector3?)
CC TypePhysical effect
"Stun"Anchors HumanoidRootPart, disables Running state.
"Freeze"Anchors root, disables all toggleable Humanoid states.
"Knockback"Applies physics impulse in knockbackDir direction.
"Root"Pins position via BodyPosition constraint without anchoring.
 
npc:ApplyCrowdControl("Stun", 2)
npc:ApplyCrowdControl("Knockback", 1, Vector3.new(0, 1, -1))
LOD Brain
SetSleeping()
LOD

Manually overrides LOD sleep state. When sleeping, all AI ticks pause and the Humanoid Running state is disabled. The LOD brain auto-sleeps NPCs beyond SleepRange and wakes them when players approach.

npc:SetSleeping(sleepState: boolean)
-- Force sleep during a cutscene
npc:SetSleeping(true)
-- Resume after cutscene
npc:SetSleeping(false)
Behaviour Tree
SetBehaviourTree()
BT

Sets the root node of the behaviour tree. The tree is ticked by LODBrain automatically at the NPC's current LOD rate.

npc:SetBehaviourTree(rootNode: BTNode)
local BT = NPCEngine.BT

npc:SetBehaviourTree(
    BT.Selector({
        BT.Sequence({
            BT.Condition(function() return npc:GetAggroTarget() ~= nil end),
            BT.Action(function()
                npc:ChangeState("Chase")
                return "Running"
            end),
        }),
        BT.Action(function()
            npc:ChangeState("Idle")
            return "Success"
        end),
    })
)
BT Node Types
BT
ConstructorDescription
BT.Sequence(children)Runs children left to right. Stops on first Failure. Returns Success only if all succeed.
BT.Selector(children)Runs children left to right. Stops on first Success. Returns Failure if all fail.
BT.Parallel(children, policy)Runs all children. Policy "all": Success if all succeed. Policy "any": Success if any succeed.
BT.Condition(fn)Returns Success if fn() returns true, Failure otherwise.
BT.Action(fn)Runs fn() which must return "Success", "Failure", or "Running".
BT.Inverter(child)Flips Success ↔ Failure from its child.
BT.Succeeder(child)Always returns Success regardless of child result.
BT.Decorator(child, fn)Wraps a child and transforms its result via a custom function.
Animation
LoadAnimation()
Animation

Loads an AnimationTrack into the named slot. Built-in slots are "idle", "walk", "run", "attack", "stun", "death". Custom slot names are supported.

npc:LoadAnimation(slotName: string, animId: string, priority: Enum.AnimationPriority?)
-- R15 default animations
npc:LoadAnimation("idle",   "rbxassetid://507766666")
npc:LoadAnimation("walk",   "rbxassetid://507777826")
npc:LoadAnimation("run",    "rbxassetid://507767714")
npc:LoadAnimation("attack", "rbxassetid://YOUR_ATTACK_ID",
    Enum.AnimationPriority.Action)
MapStateToAnimation()
Animation

Maps an FSM state name to an animation slot. When the NPC enters that state, the mapped animation plays automatically.

npc:MapStateToAnimation(stateName: string, slotName: string)
npc:MapStateToAnimation("Idle",   "idle")
npc:MapStateToAnimation("Chase",  "run")
npc:MapStateToAnimation("Attack", "attack")
PlayAnimationOverride()
Animation

Forces a specific animation slot regardless of current state or speed. Used for CC animations (stun, freeze) or special events. Call ClearAnimationOverride() to resume normal blending.

npc:PlayAnimationOverride(slotName: string)
SetDialogueScript()
Dialogue

Sets the NPC's dialogue script and activates the proximity zone. When a player enters the zone the script runs automatically.

npc:SetDialogueScript(script: DialogueScript)
npc:SetDialogueScript({
    { speaker = "Guard", text = "Halt! Who goes there?",  duration = 3 },
    { speaker = "Guard", text = "State your business.",   duration = 3 },
    { speaker = "Guard", text = "Very well. Move along.", duration = 2 },
})

npc:OnLineSpoken(function(line, index)
    -- Display in your UI here
    print(line.speaker .. ": " .. line.text)
end)
Debug Overlay
EnableDebug()
Debug

Enables a debug visualization channel. All channels auto-clean on Destroy().

npc:EnableDebug(channel: string)
ChannelWhat it draws
"stateBillboard"Current FSM state above the NPC's head.
"pathLine"Line from NPC to current path target.
"fovCone"Sight FOV arc in the XZ plane.
"aggroRadius"Circle showing hearing range.
"spatialCell"Current spatial grid cell outline.
"steeringMap"16 context steering slot directions with colour coding.
Destructor
Destroy()
Lifecycle

The final destructor. Atomically fires the current state's OnExit, stops all movement, interrupts dialogue, disconnects all network channels, fires the EventBus npc.destroyed event, and calls Janitor:Destroy() to guarantee absolute zero memory leaks.

npc:Destroy()
After calling Destroy() the NPC instance is no longer valid. Do not call any methods on it.
-- On NPC death
humanoid.Died:Connect(function()
    task.delay(3, function()
        npc:Destroy()
        model:Destroy()
    end)
end)
Object Pool
Pool.new()
Pool

Creates a pre-warmed pool of NPC instances. Allocation is deferred across frames via task.defer to avoid spawn spikes.

NPCEngine.Pool.new(config: PoolConfig) → ObjectPool
local pool = NPCEngine.Pool.new({
    template = workspace.EnemyTemplate,
    poolSize = 20,
    parent = workspace,
    onSpawn = function(npc, model)
        npc:RegisterState("Patrol", { ... })
        npc:ChangeState("Patrol")
    end,
    moveSpeed  = 14,
    sleepRange = 120,
})
SpawnWave()
Pool

Spawns multiple NPCs across spawn points, one per frame. Optional delayBetween adds extra spacing between spawns.

pool:SpawnWave(spawnPoints: { CFrame }, count: number?, delayBetween: number?)
pool:SpawnWave({
    CFrame.new(10, 5, 10),
    CFrame.new(-10, 5, 10),
    CFrame.new(0, 5, 20),
}, 15, 0.3)  -- 15 NPCs, 0.3s between each
Recycle()
Pool

Returns an NPC to the pool. Resets health, stops movement, clears blackboard, and moves the model off-world. The entry becomes available for the next SpawnOne or SpawnWave call.

pool:Recycle(npc: NPCEngineInstance)
Squad System
Squad.new()
Squad

Creates a squad that manages multiple NPCs with a shared blackboard, formation movement, and group state transitions.

NPCEngine.Squad.new(config: SquadConfig, masterJanitor: any) → GroupAI
local Janitor = require(Packages.Janitor)
local janitor = Janitor.new()

local squad = NPCEngine.Squad.new({
    name = "Alpha",
    formation = "wedge",
    leaderPolicy = "healthiest",
}, janitor)

squad:AddMember(npc1)
squad:AddMember(npc2)
squad:AddMember(npc3)

squad:SetGroupState("Chase")
squad:SetShared("target", somePlayer)
SetFormation()
Squad

Changes the squad's movement formation. Call TickFormation() every heartbeat to apply positions to followers.

squad:SetFormation(formationName: string)
FormationShape
"line"Followers trail directly behind leader.
"wedge"V-shape behind leader.
"flanks"Two columns either side of leader.
"circle"Followers surround a central point.
