Quick Start

This guide walks through building a complete working NPC from scratch in under 10 minutes.

Installation

Add NPCEngine to your wally.toml:

[dependencies]
NPCEngine = "bs_eryx/npcengine@0.1.0"

Run wally install then sync with Rojo. The engine will appear at ReplicatedStorage.Packages.NPCEngine.

Your first NPC

Place a Model in Workspace. It needs a Humanoid and a HumanoidRootPart. Then in a Script under ServerScriptService:

local ReplicatedStorage = game:GetService("ReplicatedStorage")
local NPCEngine = require(ReplicatedStorage:WaitForChild("NPCEngine"))

local npc = NPCEngine.new({
    Model = workspace.MyNPC,
    MoveSpeed = 16,
    SleepRange = 120,
})

That's it. The NPC is now live with all sub-systems initialized.

States

NPCEngine uses a Finite State Machine. Register every state before you transition to it:

npc:RegisterState("Idle", {
    OnEnter = function()
        print("Entered Idle")
    end,
    OnUpdate = function()
        -- Runs every LOD tick while in this state
        local target = npc:GetClosestPlayerInRadius(50)
        if target then
            npc:ChangeState("Chase")
        end
    end,
    OnExit = function()
        print("Exited Idle")
    end,
})

npc:RegisterState("Chase", {
    OnEnter = function() end,
    OnUpdate = function() end,
    OnExit = function()
        npc:StopMovement()
    end,
})

-- Start in Idle
npc:ChangeState("Idle")

Perception

Configure how the NPC detects players. Three senses — sight, hearing, smell — each with independent range and memory decay:

npc:ConfigurePerception({
    sightRange = 60,
    sightFOV = 90,    -- degrees
    hearingRange = 30,
    smellRange = 8,
    memoryTime = 6,     -- seconds before forgetting
})

-- React when a player is spotted
npc._perception.OnTargetSpotted:Connect(function(player, stimulus)
    print("Spotted", player.Name, "via", stimulus)
    npc:AddThreat(player, 100)
end)

-- React when a player is lost
npc._perception.OnTargetLost:Connect(function(player, lastKnownPos)
    print("Lost", player.Name, "— last seen at", lastKnownPos)
end)

Pathfinding

Navigate the NPC to any world position. Returns a cancellable Promise:

-- Simple path
npc:PathToAsync(targetPosition)
    :andThen(function()
        print("Arrived")
    end)
    :catch(function(err)
        warn("Path failed:", err)
    end)

-- Predict where a moving player will be
local playerRoot = player.Character.HumanoidRootPart
local predicted = npc:PredictTargetPosition(playerRoot, 0.3)
npc:PathToAsync(predicted):catch(function() end)

-- Stop immediately
npc:StopMovement()

Animations

Load tracks and map them to FSM states. The animation controller blends automatically:

-- Load tracks (use your own asset IDs)
npc:LoadAnimation("idle", "rbxassetid://507766666")
npc:LoadAnimation("walk", "rbxassetid://507777826")
npc:LoadAnimation("run",  "rbxassetid://507767714")

-- Map states to slots
npc:MapStateToAnimation("Idle",  "idle")
npc:MapStateToAnimation("Chase", "run")

-- Tick the controller in your update loop
npc._animController:Tick(npc._fsm._currentState)

Using Presets

Skip manual config with built-in archetypes:

local Presets = NPCEngine.Presets

-- Use a built-in archetype
local config = Presets:Apply(Presets:Get("Guard"), workspace.MyNPC)
local npc = NPCEngine.new(config)

-- Apply the preset's perception settings
local preset = Presets:Get("Guard")
if preset.perception then
    npc:ConfigurePerception(preset.perception)
end

-- Create a custom preset inheriting from Guard
Presets:Register("EliteGuard", Presets:Extend("Guard", {
    MoveSpeed = 22,
    SleepRange = 200,
    perception = { sightRange = 90, sightFOV = 130 },
}))

-- Hot-swap preset at runtime
Presets:HotSwap(npc, "Boss")

Behaviour Trees

Replace manual state transitions with a declarative behaviour tree:

local BT = NPCEngine.BT

npc:SetBehaviourTree(
    BT.Selector({

        -- Chase if we have an aggro target
        BT.Sequence({
            BT.Condition(function()
                return npc:GetAggroTarget() ~= nil
            end),
            BT.Action(function()
                npc:ChangeState("Chase")
                return "Running"
            end),
        }),

        -- Investigate last known position
        BT.Sequence({
            BT.Condition(function()
                return #npc:GetPerceivedTargets() > 0
            end),
            BT.Action(function()
                local targets = npc:GetPerceivedTargets()
                local lkp     = npc:GetLastKnownPosition(targets[1])
                if lkp then
                    npc:PathToAsync(lkp):catch(function() end)
                end
                return "Running"
            end),
        }),

        -- Fall back to idle
        BT.Action(function()
            npc:ChangeState("Idle")
            return "Success"
        end),

    })
)

Debug Overlay

Enable visual debug channels to see what the NPC is doing in Studio:

-- Individual channels
npc:EnableDebug("stateBillboard")  -- state label above head
npc:EnableDebug("fovCone")         -- sight cone arc
npc:EnableDebug("aggroRadius")     -- hearing range circle
npc:EnableDebug("pathLine")        -- line to path target
npc:EnableDebug("steeringMap")     -- 16 steering slot directions

-- Or all at once
npc:EnableAllDebug()

Cleanup

Always call Destroy() when an NPC dies or is removed. This guarantees zero memory leaks:

local humanoid = npc._humanoid
if humanoid then
    humanoid.Died:Connect(function()
        task.delay(3, function()
            npc:Destroy()
            npc._model:Destroy()
        end)
    end)
end
